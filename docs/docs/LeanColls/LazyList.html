inductive LazyList (α : Type u) :
Type u
  • nil: {α : Type u} → LazyList α
  • cons: {α : Type u} → α → LazyList α → LazyList α
  • delayed: {α : Type u} → Thunk (LazyList α) → LazyList α

Standard technique for LazyList. Necessary because LazyList is a nested inductive, but we never actually want to have different motives for the nested types.

Remark: Lean used well-founded recursion behind the scenes to define LazyList.ind
